# LBJ 模块

列车接近预警消息解析器，依据 **TB/T 3504-2018 第 9.1.2 节**「数据帧格式」。

## 消息格式

```
[5C 车次号][3C 速度][5C 公里标] = 13 个 BCD 字符
```

| 字段 | 位置 | 长度 | 说明 |
|------|------|------|------|
| 车次号 | 0-4 | 5C | BCD 数字，不足高位填空格，无效填全 `-` |
| 速度 | 5-7 | 3C | BCD 数字，不足高位填空格，无效填全 `-` |
| 公里标 | 8-12 | 5C | BCD 数字，不足高位填空格，负号在首位，无效填全 `-` |

## POCSAG 地址

| RIC | 含义 |
|-----|------|
| 1234000 | 列车接近预警 |
| 1234008 | 时钟校准 |

## 功能位

| 功能码 | 含义 |
|--------|------|
| 1 (01b) | 车次号为奇数 |
| 3 (11b) | 车次号为偶数 |

## 数据结构

```c
typedef struct {
    uint32_t ric;           // POCSAG 地址
    uint8_t  function;      // 功能码 (1=奇, 3=偶)
    char     raw_text[256]; // 原始 BCD 字符串 (13 chars)
    char     train_id[8];   // 列车号
    bool     train_odd_even;// false=奇数, true=偶数
    int16_t  speed;         // 速度 km/h
    char     km_post[8];    // 公里标
    bool     km_negative;   // 负公里标
    int16_t  rssi_dbm;      // 信号强度
} LBJ_Message_t;
```

## API 使用

```c
LBJ_Init(my_callback)       // 注册回调
LBJ_ParsePOCSAG(ric, func, text, len, rssi)  // 由 POCSAG 回调调用
```

## 设计要点

- **固定格式解析**: 不再是启发式推测，直接按 `[5C][3C][5C]` 定长切分。
- **无效标志检测**: 所有字符均为 `-` 表示该字段无效（TB/T 规定无效数据填全 D 即负号）。
- **负公里标**: 首位为 `-` 时 `km_negative=true`。
- **RIC 过滤**: 只处理 RIC=1234000 的列车接近预警消息，忽略时钟校准等其他 RIC。

## 已知问题

1. **消息长度不足 13 字符**: 如果 POCSAG 层未能完整接收 3 个消息码字，LBJ 不做解析，直接上报原始文本。
2. **车次号解析**: 当前仅提取连续数字，不判断车次号的格式规则（如字母前缀等）。如果实际铁路车次号格式更复杂，需扩展。
