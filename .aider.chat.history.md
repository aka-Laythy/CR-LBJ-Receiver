```c
    if (USART_GetITStatus(USART1, USART_IT_RXNE) != RESET) {

        __ring_push(&gps_rx_ring, byte, GPS_RX_BUF_SIZE);

        if (byte == '$') {

            gps_line_buf[gps_line_idx++] = byte;
            if (byte == '\n') {
                gps_line_buf[gps_line_idx] = '\0';
                gps_line_ready = true;
                gps_line_idx = 0;

            gps_line_idx = 0;
```

- 行缓冲区 `gps_line_buf` 大小为 **128 字节**（`GPS_LINE_BUF_SIZE`）。  
- 若 NMEA 语句长度超过 128 字节（极少数情况），会导致溢出并丢失该行数据。  
- 如需支持更长语句，可增大 `GPS_LINE_BUF_SIZE`。

### 2. 解析仅支持 GGA 和 RMC
- 当前仅解析 `$GPGGA` / `$GNGGA` 和 `$GPRMC` / `$GNRMC` 语句。

- 时间转换仅处理 UTC+8 跨日情况，未考虑跨月/跨年的复杂边界（如 23:59:59 UTC → 07:59:59 次日）。

- 对于高精度应用，建议使用更完善的日期库或自行扩展。

### 4. 时间戳计算未考虑闰秒
- `timestamp` 字段基于简单天数累加，未考虑闰秒。  
- 对于大多数应用（误差 < 1 秒）可接受。

### 5. 中断优先级
- USART1 中断优先级为 `PreemptionPriority=1, SubPriority=1`。  
- 若与其他高优先级中断（如 SysTick）冲突，需调整优先级分组或数值。

### 6. 调试输出
- 当前代码未在解析成功/失败时输出调试信息。  
- 如需调试，可在 `GPS_ParseNMEA()` 中添加 `printf` 或通过 BLE 发送日志。

### 7. 内存占用
- 环形缓冲区 `gps_rx_ring` 占用 512 字节，`gps_tx_ring` 占用 128 字节。  
- 行缓冲区 `gps_line_buf` 占用 128 字节。

## 潜在问题与注意事项

### 1. 行缓冲区大小限制
- 行缓冲区 `gps_line_buf` 大小为 **128 字节**（`GPS_LINE_BUF_SIZE`）。  
- 若 NMEA 语句长度超过 128 字节，会导致溢出并丢失该行数据。  
- 如需支持更长语句，可增大 `GPS_LINE_BUF_SIZE`。

### 2. 帧间隔超时机制
- 当前代码仅依赖 `'\n'` 作为帧结束标志，未实现超时检测。  
- 若模块输出 1PPS（每秒一帧），100ms 超时是合理的帧间隔判断。  
- 如需更可靠的帧边界检测，建议在 `USART1_IRQHandler` 中添加超时逻辑。

### 3. 解析仅支持 GGA 和 RMC
- 当前仅解析 `$GPGGA` / `$GNGGA` 和 `$GPRMC` / `$GNRMC` 语句。  
- 若模组输出其他语句（如 `$GPZDA`），将被忽略。  
- 如需支持更多语句，需在 `GPS_ParseNMEA()` 中添加对应解析函数。

### 4. 北京时间转换
- 时间从 RMC 语句中提取，UTC 时间转换为北京时间（UTC+8）。  
- 跨日处理已实现，使用北京时间年份判断闰年。  
- 时间戳字段保持 UTC 时间。

### 5. 时间戳计算未考虑闰秒
- `timestamp` 字段基于简单天数累加，未考虑闰秒。  
- 对于大多数应用（误差 < 1 秒）可接受。

### 6. 中断优先级
- USART1 中断优先级为 `PreemptionPriority=1, SubPriority=1`。  
- 若与其他高优先级中断（如 SysTick）冲突，需调整优先级分组或数值。

### 7. 调试输出
- 当前代码未在解析成功/失败时输出调试信息。  
- 如需调试，可在 `GPS_ParseNMEA()` 中添加 `printf` 或通过 BLE 发送日志。

### 8. 内存占用
- 环形缓冲区 `gps_rx_ring` 占用 512 字节，`gps_tx_ring` 占用 128 字节。  
- 行缓冲区 `gps_line_buf` 占用 128 字节。  
- 全局结构体 `gps_data_buffer` 占用约 40 字节。  
- 总计约 808 字节 RAM，在 CH32V00X 系列中可接受。

## 关于溢出风险的详细说明

### 无符号减法原理

```c
uint32_t now = systick_get_ms();    // 当前时间
uint32_t elapsed = now - last_ms;   // 经过时间
```

- 如果 `now >= last_ms`，`elapsed = now - last_ms`，正常。
- 如果 `now < last_ms`（溢出后），`elapsed = now - last_ms` 在无符号运算中等于 `(2^32 + now) - last_ms`，结果仍然是正确的经过时间。

### 示例

假设 `last_ms = 4,294,967,290`（溢出前 5ms），`now = 10`（溢出后 10ms）：

```
elapsed = 10 - 4,294,967,290
        = (4,294,967,296 + 10) - 4,294,967,290
        = 16 ms
```

正确！

### 防突发逻辑

```c
if (elapsed >= (2 * tasks[i].interval_ms)) {
```
