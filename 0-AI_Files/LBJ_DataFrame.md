## 2. 列车接近预警消息码（§9.1.2.4，表 10）

### 2.1 基本帧格式（POCSAG）

```
前置码(576bit) | SC(同步字) | 帧0 | 帧1 | ... | 帧7 | SC | ...
```

每帧 = 2 个码字（64bit），地址码字在第 0~7 帧中分配，消息码字紧随。

### 2.2 字段定义

OCR 文本显示表 10 为：

```
序号  名称     比特数  定义
────────────────────────────────────────────────────────
1     车次号   20     BCD 编码。不足五位高位填充空格符。
                      数据无效时填充五位 "D"（即 0xD 负号）
      [分隔符]  4     [推测为空格符，OCR 字符 "e" / "("]
2     速度     12     BCD 编码。不足三位高位填充空格符。
                      数据无效时填充三位 "D"
      [分隔符]  4     [空格符]
3     公里标   20     BCD 编码。不足五位高位填充空格符，
                      负公里标首位填充负号。数据无效时填充五位 "D"
```

**总比特数：** 20 + 4 + 12 + 4 + 20 = **60 bit = 15 × 4-bit 字符**

### 2.3 POCSAG 字符编码（表 12）

| 4-bit | 十进制码 | 字符 | 说明 |
|-------|---------|------|------|
| `0000` | 0 | `0` | |
| `0001` | 1 | `1` | |
| `0010` | 2 | `2` | |
| `0011` | 3 | `3` | |
| `0100` | 4 | `4` | |
| `0101` | 5 | `5` | |
| `0110` | 6 | `6` | |
| `0111` | 7 | `7` | |
| `1000` | 8 | `8` | |
| `1001` | 9 | `9` | |
| `1010` | 10 | `A` | 备用（当前未使用） |
| `1011` | 11 | `B` | OCR 不清晰 |
| `1100` | 12 | `C` | **空格符**（用于字段间分隔或高位填充） |
| `1101` | 13 | `D` | **负号 `-`**（无效数据标记、负公里标符号） |
| `1110` | 14 | `E` | `]` 符 |
| `1111` | 15 | `F` | `[` 符 |

### 2.4 关于分隔符的说明

**⚠ 推测标注：** 表 10 的 OCR 结果中出现了两行无序号（空格符）行，分别位于车次号与速度之间、速度与公里标之间，各 4 bit，值为空格（POCSAG 编码 `1100`）。

若分隔符确实存在，则完整消息码格式为：

```
[5C 车次号][1C 空格][3C 速度][1C 空格][5C 公里标] = 15 字符
```

字段偏移应为：

| 字段 | 起始位置 | 长度 |
|------|---------|------|
| 车次号 | 0 | 5 |
| 空格 | 5 | 1 |
| 速度 | 6 | 3 |
| 空格 | 9 | 1 |
| 公里标 | 10 | 5 |

当前代码实现（`LBJ.c`）按 13 字符处理（无分隔符），偏移为：

| 字段 | 起始位置 | 长度 |
|------|---------|------|
| 车次号 | 0 | 5 |
| 速度 | 5 | 3 |
| 公里标 | 8 | 5 |

**两种偏移是否兼容：** 若真实消息带分隔符，当前代码提取速度（取第 5~7 位）时会将分隔空格当作速度的第一位，导致速度值错位。公里标偏移同样错误。建议通过实际 SDR 录制的 raw 消息验证分隔符是否存在，再决定采用 13 字符还是 15 字符解析。

## 3. 时钟校准消息（§9.1.2.5，表 11）

| 序号 | 名称 | 比特数 | 定义 |
|------|------|--------|------|
| 1 | 起始标志 | 4 | `-`（负号 0xD） |
| 2 | 时 | 8 | BCD，2 字符（00~23） |
| 3 | 分 | 8 | BCD，2 字符（00~59） |

格式：`-HHMM`，共 5 字符。通过 RIC = `1234008` 识别。

---

## 4. 报警信道数据帧（§9.2）—— 与本项目无关

采用 **FFSK** 调制（非 POCSAG），用于道口报警/施工防护报警，包含：

| 字段 | 字节数 | 说明 |
|------|--------|------|
| 信息类别 | 1 | `03H`=道口报警, `04H`=解除, `05H`=施工报警, `06H`=施工解除 |
| 公里标 | 3 | 24-bit 有符号整数，单位 `米` |
| 信息发送时间 | 4 | 压缩 BCD：秒(6b)+分(6b)+时(5b)+日(5b)+月(4b)+年(6b) |
| 线路名称 | 8 | **用内码表示**（非 ASCII，非 UTF-8，是铁路系统专用编码） |
| 预留 | 1 | `FFH` |

此协议不走 POCSAG，当前项目不涉及，仅作参考。
