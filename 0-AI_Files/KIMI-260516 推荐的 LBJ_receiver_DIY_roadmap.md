# DIY 便携式 LBJ/POCSAG 接收器学习与实现路线图

> 目标：以学习和个人实验为目的，做出一个基于射频接收芯片和 MCU 的便携式 LBJ/POCSAG 接收器。  
> 原则：先跑通最小链路，再补理论，再画 PCB；避免一开始追求“最终产品”。

## 0. 边界与安全

这个项目只做接收，不做发射，不做干扰，不作为铁路安全保障设备使用。

你要做的是一个学习型接收器：

```text
天线 -> 射频接收芯片 -> 原始 bit 流 -> MCU 解码 POCSAG -> 解析 LBJ 文本 -> 显示/蜂鸣/记录
```

第一阶段不要追求：

- 极限距离
- 超低功耗
- 完整外壳
- 商用品质
- 裸 SX1276/SX1262 射频设计
- 复杂菜单、日志、联网、GPS

第一阶段只追求一件事：**稳定收到并解出消息**。

## 1. 总体路线

推荐路线：

```text
阶段 1：复现开源模块方案
阶段 2：理解 FSK/POCSAG/LBJ 的最小理论
阶段 3：把解码逻辑移植到自己熟悉的 MCU
阶段 4：做模块版 PCB
阶段 5：优化接收性能和便携体验
阶段 6：再考虑 SX1262/LLCC68 或裸芯片方案
```

不推荐路线：

```text
先完整学习通信原理 -> 直接画最终 PCB -> 一次性做外壳、电池、显示、日志、低功耗
```

这样容易同时踩射频、固件、协议、电源、结构五类坑，定位问题会很困难。

## 2. 阶段 1：复现最小原型

### 目标

用开发板和射频模块搭出可以观察原始信号的原型。

推荐硬件：

- 你熟悉的 MCU 开发板，例如 CH32V307
- SX1276 868/915 MHz 模块，或 CC1101 820/915 MHz 可用模块
- 821 MHz 附近可用天线
- USB 串口
- 逻辑分析仪

注意：不要买 433 MHz 匹配版本的模块。LBJ 相关频点在 821 MHz 附近，模块前端匹配频段必须合适。

### 要学什么

- MCU SPI 基本读写
- GPIO 输入
- 外部中断
- 定时器或边沿采样
- 串口打印调试
- 逻辑分析仪抓 GPIO 波形

### 学到什么程度算够

满足以下条件就够：

- 能通过 SPI 读出射频芯片版本号或状态寄存器
- 能配置射频芯片进入 FSK 接收模式
- 能从 DIO 引脚看到 data 或 clock 波形
- 能用逻辑分析仪确认 MCU 采样到的 bit 和 DIO 波形一致
- 能在串口持续打印 RSSI、同步状态或原始 bit 统计

不需要达到：

- 熟练写完整射频驱动框架
- 完全理解芯片内部 PLL、混频器、LNA、AGC 原理
- 自己设计射频前端

### 本阶段产出

```text
开发板 + 射频模块 + 串口输出
```

串口至少能输出类似：

```text
radio ok
rssi: -83 dBm
sync: searching
sync: found
raw codeword: 7A89C197
```

## 3. 阶段 2：补最小通信理论

### 目标

理解这个项目里“信号如何变成 bit”，不需要学成通信专业研究深度。

### 要学什么

#### 3.1 载波与调制

需要理解：

- 无线信号有一个中心频率
- 载波上可以承载信息
- 调制就是把信息映射到载波的某种变化上

学到什么程度算够：

- 能解释“821.2375 MHz 是中心频率”
- 能解释“MCU 不直接处理 821 MHz 射频，而是处理射频芯片解调后的 bit”
- 能区分“射频频率”和“数据速率”

不需要达到：

- 推导傅里叶变换
- 推导调制频谱
- 熟练计算链路预算

#### 3.2 FSK / 2FSK / FFSK

需要理解：

- FSK 是用频率变化表示数字信息
- 2FSK 是两个频率表示 0 和 1
- 频偏是相对中心频率偏高/偏低的距离
- 接收带宽太窄会丢信号，太宽会引入更多噪声

学到什么程度算够：

- 能解释“高一点的频率代表 0 或 1，低一点的频率代表另一个值”
- 能看懂参数：`1200 bps`、`4.5 kHz deviation`、`58 kHz RX bandwidth`
- 能理解为什么有些芯片需要 bit 反相
- 能知道 SX1276 此处不是跑 LoRa，而是在 FSK 模式下当数字接收前端

不需要达到：

- 自己从 IQ 数据手写完整 FSK 解调器
- 推导最佳判决门限
- 推导误码率公式

#### 3.3 RSSI、SNR、误码

需要理解：

- RSSI 是接收信号强度估计
- RSSI 高不一定等于一定能解码
- 频偏、带宽、天线、供电噪声都会影响误码

学到什么程度算够：

- 能记录每次成功/失败解码时的 RSSI
- 能判断“有信号但不同步”和“完全没信号”是两类问题
- 能知道调试时要同时看 RSSI、同步字、码字校验结果

不需要达到：

- 做严格的接收灵敏度测试
- 搭专业射频测试环境

## 4. 阶段 3：学习 POCSAG 协议

### 目标

从 bit 流中识别 POCSAG 消息，并解析出 numeric 文本。

### 要学什么

POCSAG 最少要掌握：

- 前导码：连续 `101010...`
- 同步字：`0x7CD215D8`
- 每个码字 32 bit
- 地址码字和消息码字的区别
- IDLE 码字：`0x7A89C197`
- 偶校验
- BCH 纠错
- numeric 编码

### 学到什么程度算够

满足以下条件就够：

- 能从连续 bit 流里滑动查找 `0x7CD215D8`
- 找到同步字后，能每 32 bit 切成一个 codeword
- 能识别地址码字和消息码字
- 能把 message codeword 中的 20 bit payload 转成 numeric 字符
- 能处理 IDLE 码字
- 能做至少 1 bit 错误纠正，最好支持 2 bit 错误检测/纠正

不需要达到：

- 写完整通用 POCSAG 寻呼机协议栈
- 支持所有速率、所有编码类型、所有寻呼格式
- 一开始就支持复杂 BCH 优化

### 本阶段产出

建议先写一个 PC 端或 MCU 端小测试：

```text
输入：一组已知 POCSAG codeword
输出：地址、功能码、numeric 文本
```

再接入实时 bit 流。

## 5. 阶段 4：理解 LBJ 消息格式

### 目标

把 POCSAG numeric 文本解释成你关心的列车接近预警信息。

### 要学什么

重点掌握：

- LBJ 使用固定地址发送特定业务消息
- 消息内容是 numeric 字符串
- 基础消息通常包含列车号、速度、公里标
- 扩展消息可能包含机车型号、机车号、端位、线路、经纬度等
- 同一列车信息可能分两条消息出现，需要合并

### 学到什么程度算够

满足以下条件就够：

- 能识别目标地址，例如列车接近预警消息地址
- 能解析基础消息：列车号、速度、公里标
- 能把公里标小数点位置处理正确
- 能识别时间同步消息
- 能把连续两条相关消息合并成一次显示
- 遇到未知字段时保留 raw 文本，不要丢弃

不需要达到：

- 建完整机车型号数据库
- 支持所有线路的扩展字段
- 一开始就做地图显示

## 6. 阶段 5：移植到自己的 MCU

### 目标

用 CH32V307 或你熟悉的 MCU 跑通完整链路。

### 推荐软件结构

```text
radio_sx1276.c      射频芯片寄存器配置、RSSI、模式切换
bit_capture.c       DIO clock/data 采样、环形缓冲
pocsag_decoder.c    同步字、码字、BCH、numeric 解码
lbj_parser.c        LBJ 字段解析
app.c               显示、蜂鸣、按键、日志
```

### 要学什么

- 状态机写法
- 中断里只做轻量工作
- 环形缓冲区
- 固件模块边界
- 错误计数和超时恢复
- 串口日志分级

### 学到什么程度算够

满足以下条件就够：

- 中断里只采样 bit 或搬运数据，不在中断里解析完整消息
- 主循环可以持续运行，不因一次坏包卡死
- 射频模块掉线或长时间无同步时能自动重新初始化
- 解码失败时能打印失败原因：无同步、BCH 失败、地址不匹配、消息为空
- 同一份 POCSAG/LBJ 解析逻辑可以用离线测试数据验证

不需要达到：

- RTOS
- DMA 全链路优化
- 复杂任务调度框架
- 大规模可配置插件式架构

## 7. 阶段 6：做第一版 PCB

### 目标

做一块能长期稳定工作的模块版 PCB。

### 第一版推荐硬件

- MCU：CH32V307 或你熟悉的芯片
- 射频：SX1276 868/915 MHz 模块
- 显示：0.96 OLED 或小尺寸 LCD
- 提示：蜂鸣器或震动马达
- 电源：锂电池 + 充电 IC + 3.3V LDO
- 调试：USB 串口、SWD、测试点
- 天线：SMA 或 IPEX

### PCB 要学什么

- 50Ω 走线的基本概念
- RF 走线短、直、少过孔
- 射频区完整地平面
- 天线周围净空
- 数字噪声远离天线和射频模块
- 电源去耦
- 锂电池保护和充电安全
- 测试点布局

### 学到什么程度算够

满足以下条件就够：

- 能用阻抗计算器算出大致 50Ω 线宽
- 模块天线口到 SMA/IPEX 的走线短且连续
- RF 走线下方有完整参考地
- 天线附近没有乱放铜皮、屏幕排线、蜂鸣器、电感
- 射频模块电源旁边有足够去耦
- MCU 的 SPI、DIO、复位、供电都有测试点
- 充电和电池部分不会反接、过流或短路

不需要达到：

- 裸芯片射频匹配
- VNA 调天线
- 多层高频板专业设计
- 量产级 EMC 设计

## 8. 阶段 7：调试方法

### 目标

出现问题时能定位在哪一层，而不是盲目改参数。

### 分层排查

#### 8.1 射频层

检查：

- 模块频段是否正确
- 天线是否合适
- 频率是否配置为目标频点
- RSSI 是否有明显变化
- 供电是否稳定

够用标准：

- 能判断“有没有收到东西”
- 能比较不同天线、不同位置的 RSSI 差异

#### 8.2 bit 流层

检查：

- DIO clock 是否稳定
- data 是否随信号变化
- MCU 是否漏采样
- bit 是否需要反相

够用标准：

- 能用逻辑分析仪确认 MCU 采样逻辑正确
- 能统计单位时间内收到多少 bit

#### 8.3 POCSAG 层

检查：

- 是否找到同步字
- 同步字是否反相
- codeword 是否 32 bit 对齐
- BCH/偶校验失败率

够用标准：

- 能区分“找不到同步”和“找到同步但码字坏”
- 能把失败样本保存下来离线分析

#### 8.4 LBJ 解析层

检查：

- 地址是否正确
- numeric 字符是否映射正确
- 基础消息字段是否错位
- 扩展消息是否需要合并

够用标准：

- raw 消息和解析结果同时保留
- 解析错了可以通过 raw 文本回溯

## 9. 阶段 8：优化方向

在第一版稳定解码后再优化。

### 可以优化

- 更好的 821 MHz 天线
- 更低噪声的 3.3V 电源
- 更准确的晶振或 TCXO
- 自动频偏补偿
- 更稳健的 BCH 纠错
- 更好的消息合并逻辑
- 低功耗休眠
- 外壳和便携体验

### 可以考虑替换芯片

- SX1262：功耗和抗干扰更好，但驱动不同
- LLCC68：性价比高，但也不是 SX1276 寄存器级平替
- CC1101：资料多，FSK 场景成熟，但频段模块和前端匹配要注意

替换芯片前应满足：

- 你已经有一版 SX1276 或 CC1101 模块原型稳定工作
- POCSAG/LBJ 解码模块已经和 radio 驱动解耦
- 你能用同一套测试数据验证新旧芯片输出是否一致

## 10. 推荐学习顺序

### 第 1 周：看懂系统链路

目标：

- 明白 SX1276 此处不是 LoRa，而是 FSK 接收
- 明白 POCSAG 是 bit 流上的协议
- 明白 LBJ 是 POCSAG numeric 消息上的业务格式

够用标准：

- 能画出完整链路图
- 能解释每一层输入输出是什么

### 第 2-3 周：搭开发板原型

目标：

- SPI 能读写射频芯片
- DIO 能进 MCU
- 串口能打印状态

够用标准：

- 能看到 RSSI
- 能看到 clock/data 或 FIFO 数据

### 第 4-5 周：POCSAG 离线解码

目标：

- 写出或移植 POCSAG codeword 解析
- 能从测试数据解出 numeric 文本

够用标准：

- 能识别同步字
- 能切 32 bit codeword
- 能解析地址和消息

### 第 6-7 周：实时解码

目标：

- 把 bit capture 和 POCSAG decoder 接起来
- 能在串口看到实时消息

够用标准：

- 不重启能连续运行
- 成功消息和失败原因都有日志

### 第 8-10 周：第一版 PCB

目标：

- 做模块版便携样机

够用标准：

- PCB 上电正常
- 射频模块可通信
- 屏幕/蜂鸣器可用
- 实际能接收并解码

## 11. 最小知识闭环

如果只记住一条主线，就是：

```text
FSK 负责把无线信号变成 0/1
POCSAG 负责把 0/1 组织成地址和消息
LBJ 负责定义消息里的列车信息含义
MCU 负责采样、解码、显示和提示
PCB 负责让这些东西稳定供电、连接和接收
```

做到下面这些，就说明你已经具备做出第一版的能力：

- 能配置 SX1276/CC1101 进入目标 FSK 接收参数
- 能稳定采样 bit 流
- 能找到 POCSAG 同步字
- 能解析 codeword
- 能从 numeric 文本中提取列车号、速度、公里标
- 能把结果显示出来
- 能用日志定位失败原因

这时再继续深入通信原理、射频电路和低功耗设计，会更有效，因为你已经有了一个可验证的实物平台。
